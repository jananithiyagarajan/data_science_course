# Constraints

In SQL, constraints are rules applied to columns or tables to ensure the accuracy, reliability, and integrity of the data stored in the database.

## Types of SQL Constraints

**1. NOT NULL**

Ensures the column cannot store NULL values.

**2. UNIQUE**

Ensures no duplicate values in a column.

**3. PRIMARY KEY**

Combination of NOT NULL + UNIQUE. Identifies each row uniquely.

**4. FOREIGN KEY**

Maintains relationship between two tables; ensures **referential integrity**.

**5. CHECK**

Ensures values meet a specific condition.

**6. DEFAULT**

Applies a default value if no value is provided.

**Summary Table**

| Constraint  | Purpose                                    |
| ----------- | ------------------------------------------ |
| NOT NULL    | Value cannot be empty                      |
| UNIQUE      | No duplicates allowed                      |
| PRIMARY KEY | Unique + Not Null                          |
| FOREIGN KEY | Links tables                               |
| CHECK       | Validates conditions                       |
| DEFAULT     | Adds default value                         |
| INDEX       | Speeds up search (not a strict constraint) |

**Step 2: Practice NOT NULL and UNIQUE**


The `NOT NULL` constraint prevents empty values, while `UNIQUE` ensures no duplicate entries.

### Create a 'Users' Table
```sql
CREATE TABLE users (
    user_id INT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    phone VARCHAR(15) UNIQUE
);
```

**Step 3: Practice PRIMARY KEY and FOREIGN KEY**

`FOREIGN KEY` links rows between two tables, ensuring **Referential Integrity**.

### Create a 'Profiles' Table linked to 'Users'

```sql
CREATE TABLE profiles (
    profile_id INT PRIMARY KEY,
    user_id INT NOT NULL,
    bio TEXT,
    FOREIGN KEY (user_id) REFERENCES users(user_id)
);
```

**Step 4: Practice CHECK and DEFAULT**

`CHECK` validates a condition, and `DEFAULT` provides a fallback value.

### Create a 'Products' Table

```sql
CREATE TABLE products (
    product_id INT PRIMARY KEY,
    product_name VARCHAR(100) NOT NULL,
    price DECIMAL(10,2) CHECK (price > 0),
    stock INT DEFAULT 0,
    category VARCHAR(50) DEFAULT 'General'
);
```

**1. Table: Customers**

```
CREATE TABLE Customers (
    CustomerID INT PRIMARY KEY,                  -- PRIMARY KEY
    FullName VARCHAR(100) NOT NULL,              -- NOT NULL
    Email VARCHAR(100) UNIQUE NOT NULL,          -- UNIQUE + NOT NULL
    Age INT CHECK (Age >= 18),                   -- CHECK Constraint
    Status VARCHAR(20) DEFAULT 'Active'          -- DEFAULT
);
```

**2. Table: Accounts**

```
CREATE TABLE Accounts (
    AccountNo INT PRIMARY KEY,                    -- PRIMARY KEY
    CustomerID INT NOT NULL,                      -- NOT NULL
    AccountType VARCHAR(20) DEFAULT 'Savings',    -- DEFAULT
    Balance DECIMAL(10,2) CHECK (Balance >= 0),   -- CHECK
    FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID)   -- FOREIGN KEY
);
```

**3. Table: Transactions**

```
CREATE TABLE Transactions (
    TxnID INT PRIMARY KEY,                           -- PRIMARY KEY
    AccountNo INT NOT NULL,                          -- NOT NULL
    Amount DECIMAL(10,2) NOT NULL CHECK (Amount > 0),-- CHECK
    TxnType VARCHAR(10) CHECK (TxnType IN ('Credit', 'Debit')), 
    FOREIGN KEY (AccountNo) REFERENCES Accounts(AccountNo)   -- FOREIGN KEY
);
```

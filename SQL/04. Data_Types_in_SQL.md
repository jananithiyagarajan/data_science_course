## 1. Numeric Data Types

Used for storing numbers.

| Data Type    | Description                  | Example    |
| ------------ | ---------------------------- | ---------- |
| INT          | Whole numbers                | 10         |
| SMALLINT     | Smaller range of integers    | 120        |
| BIGINT       | Very large integers          | 9000000000 |
| DECIMAL(p,s) | Exact numbers with precision | 25.50      |
| NUMERIC(p,s) | Same as DECIMAL              | 100.75     |
| FLOAT        | Approx numeric value         | 12.345     |
| REAL         | Approx float                 | 3.14       |

**Use cases:** salary, price, age, quantities.

**Step 1: Numeric Data Types (INT, DECIMAL, FLOAT)**

Numeric types are used for everything from IDs to financial data.

```
CREATE DATABASE datatypes_practice;
USE datatypes_practice;
```

```
CREATE TABLE products (
    product_id INT PRIMARY KEY,
    product_name VARCHAR(100),
    price DECIMAL(10,2),      -- Exact decimal (10 digits total, 2 after decimal)
    weight FLOAT,             -- Approximate decimal
    stock_count SMALLINT      -- Smaller integer for efficiency
);
```

## 2. String / Character Data Types

Used to store text or characters.

| Data Type  | Description            | Example         |
| ---------- | ---------------------- | --------------- |
| CHAR(n)    | Fixed-length string    | 'ABCD'          |
| VARCHAR(n) | Variable-length string | 'John'          |
| TEXT       | Large text data        | Notes, comments |

**Use cases:** names, emails, descriptions.

**Step 2: String / Character Data Types (CHAR, VARCHAR, TEXT)**

Understanding the difference between fixed and variable length strings.

```
CREATE TABLE users (
    user_id INT PRIMARY KEY,
    country_code CHAR(3),      -- Fixed length (always 3 chars)
    username VARCHAR(50),     -- Variable length (up to 50 chars)
    bio TEXT                  -- For large amounts of text
);
```

## 3. Date and Time Data Types

Store dates and time values.

| Data Type | Description              | Example             |
| --------- | ------------------------ | ------------------- |
| DATE      | YYYY-MM-DD               | 2025-02-10          |
| TIME      | HH:MM:SS                 | 14:30:00            |
| DATETIME  | Date + time              | 2025-02-10 14:30:00 |
| TIMESTAMP | Auto-stores current time | 2025-02-10 14:30:00 |
| YEAR      | Year value               | 2025                |

**Use cases:** created_at, login time, transaction dates.

**Step 3: Date and Time Data Types (DATE, DATETIME, TIMESTAMP)**

Understanding the difference between fixed and variable length strings.

```
CREATE TABLE users (
```

## 4. Boolean Data Types

| Data Type      | Description   | Example |
| -------------- | ------------- | ------- |
| BOOLEAN / BOOL | True or False | TRUE    |

**Use cases:** status flags, switches.

**Step 4: Advanced Data Types (JSON, BOOLEAN)**

Modern SQL databases support structured formats like JSON and simple flags.

```
CREATE TABLE settings (
    id INT PRIMARY KEY,
    app_name VARCHAR(50),
    is_active BOOLEAN,             -- TRUE or FALSE
```

## 5. Binary Data Types

Store raw binary data like files, images.

| Data Type                  | Description            |
| -------------------------- | ---------------------- |
| BINARY                     | Fixed-length binary    |
| VARBINARY                  | Variable-length binary |
| BLOB (Binary Large Object) | Images, files, videos  |

**Step 5: Binary Data Types**

Binary data types are used to store raw binary data such as files, images, and encryption keys.

## 6. Spatial / Special Data Types

(Used in advanced databases)

| Data Type | Description                             |
| --------- | --------------------------------------- |
| GEOMETRY  | Shape data                              |
| JSON      | Stores JSON objects (MySQL, PostgreSQL) |

Step 6: Spatial / Special Data Types

Used in advanced databases for geographical data or structured storage.

```
CREATE TABLE locations (
    id INT PRIMARY KEY,
    place_name VARCHAR(100),
    coordinates GEOMETRY,         -- Stores shape data (Points, Polygons)
    extra_details JSON            -- Stores JSON objects
);
```

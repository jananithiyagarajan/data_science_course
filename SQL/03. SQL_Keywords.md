## 1. DDL – Data Definition Language

Used to **create, change, or delete** database structures.

| Keyword  | Purpose                                                     |
| -------- | ----------------------------------------------------------- |
| CREATE   | Create database objects (table, view, index)                |
| ALTER    | Modify existing objects                                     |
| DROP     | Delete objects                                              |
| TRUNCATE | Remove all rows quickly (cannot be rolled back in some DBs) |
| RENAME   | Rename a table                                              |

**Step 1: DDL — Data Definition Language (ALTER, RENAME, DROP)**

We already know `CREATE`. Now let's practice modifying existing structures.


## 2. DML – Data Manipulation Language

Used to **insert, modify, read** data inside tables.

| Keyword | Purpose                 |
| ------- | ----------------------- |
| SELECT  | Retrieve data           |
| INSERT  | Add new data            |
| UPDATE  | Change existing data    |
| DELETE  | Remove specific records |

---

## 5. TCL – Transaction Control Language

Used to manage **transactions**.

| Keyword         | Purpose                                   |
| --------------- | ----------------------------------------- |
| COMMIT          | Save all changes permanently              |
| ROLLBACK        | Undo changes made in a transaction        |
| SAVEPOINT       | Create a checkpoint to rollback partially |
| SET TRANSACTION | Define transaction settings               |

---

# 6. Constraints Keywords

Used to maintain **data integrity**.

| Keyword     | Purpose                 |
| ----------- | ----------------------- |
| PRIMARY KEY | Uniquely identifies row |
| FOREIGN KEY | Links tables            |
| UNIQUE      | Prevents duplicates     |
| NOT NULL    | Prevents null values    |
| CHECK       | Validates conditions    |
| DEFAULT     | Adds default value      |

---

# 7. Filtering and Sorting Keywords

| Keyword        | Purpose                       |
| -------------- | ----------------------------- |
| WHERE          | Filter rows                   |
| ORDER BY       | Sort results                  |
| GROUP BY       | Group rows                    |
| HAVING         | Filter results after grouping |
| DISTINCT       | Remove duplicates             |
| BETWEEN        | Range condition               |
| LIKE           | Pattern matching              |
| IN             | Match multiple values         |
| AND / OR / NOT | Logical conditions            |

---

# 8. Joins Keywords

Used to combine data from multiple tables.

| Keyword    | Purpose                             |
| ---------- | ----------------------------------- |
| INNER JOIN | Match rows in both tables           |
| LEFT JOIN  | All rows from left table + matches  |
| RIGHT JOIN | All rows from right table + matches |
| FULL JOIN  | All rows from both tables           |
| CROSS JOIN | Cartesian product                   |

---

# 9. Other Important Keywords

| Keyword     | Purpose                              |
| ----------- | ------------------------------------ |
| AS          | Give alias to columns or tables      |
| LIMIT / TOP | Restrict number of rows              |
| EXISTS      | Check if a subquery returns data     |
| ANY / ALL   | Compare values in subquery           |
| UNION       | Combine results removing duplicates  |
| UNION ALL   | Combine results including duplicates |
| CASE        | Conditional logic inside query       |
